A routing table determines the path a packet follows to reach its destination network. When a destination IP address exists outside the local subnet, the packet is forwarded to the default gateway, which is the router responsible for directing traffic beyond the LAN. If both the source and destination IP addresses share the same subnet, the packet can be delivered directly without router involvement. However, when the destination is located on a different subnet, the router forwards the packet toward the appropriate network.

The VM’s IPv4 address is 10.12.27.133. This address is not globally unique because it falls within a private IPv4 range. As a result, another device on a separate private network could legally use the same address without conflict.

IP Address Ranges
There are three designated private IPv4 ranges:
10.0.0.0 – 10.255.255.255
172.16.0.0 – 172.31.255.255
192.168.0.0 – 192.168.255.255
Since 10.12.27.133 falls within the 10.0.0.0/8 block, it is classified as a private IP address.
A device may appear to have two IP addresses because it uses a private address internally and a public address externally. The private address is typically assigned by the router and is used only within the LAN. The public address represents the network on the broader internet.
